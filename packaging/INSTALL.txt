Truck Packer — install locations
================================

Build artifacts (from this repo)
--------------------------------
  macOS DMG:   run packaging/build_mac_dmg.sh  → packaging/TruckPacker-<version>-macOS.dmg
  macOS PKG:   run packaging/build_mac_pkg.sh  → packaging/TruckPacker-<version>-macOS.pkg
  Windows EXE: install Inno Setup 6, then run packaging/build_win_installer.ps1
               → packaging/TruckPacker-<version>-Windows-Setup.exe (and a .zip fallback)
  Windows ZIP: same script always builds packaging/TruckPacker-<version>-Windows.zip

GitHub: use "Releases" to attach DMG/EXE/ZIP — not "Packages" (Packages is for npm/Docker/NuGet-style registries, not plug-in installers).

CI: pushing a git tag matching v* runs .github/workflows/release.yml and uploads macOS DMG/PKG plus Windows ZIP/EXE to that tag's Release (bump project(VERSION) in CMakeLists.txt first so artifact names match the release).


This folder was produced by the packaging/stage_plugins.cmake script (or an installer built from it).

macOS
-----
Copy (merge) into the system or user plug-in folders:

  Library/Audio/Plug-Ins/VST3/Truck Packer.vst3
  Library/Audio/Plug-Ins/Components/Truck Packer.component   (AU)

Typical user paths (no admin password):

  ~/Library/Audio/Plug-Ins/VST3/
  ~/Library/Audio/Plug-Ins/Components/

After copying, restart your DAW or rescan plug-ins.

Windows
-------
Copy the entire folder:

  VST3/Truck Packer.vst3

into the 64-bit VST3 directory (common choice):

  C:\Program Files\Common Files\VST3\

The Microsoft Edge WebView2 runtime must be installed (Windows 11 includes it; on Windows 10 install "Evergreen WebView2 Runtime" from Microsoft if the embedded browser is blank).

Loop audio (inspirational music)
--------------------------------
A **videoplayback.mp4** is **embedded in the plug-in** at build time from `resources/videoplayback.mp4` (replace that file and rebuild to change it).

Optional overrides (first match wins after the embedded decode fails):

  TRUCK_PACKER_LOOP   full path to any supported file
  ~/Downloads, Desktop, Documents, Music, or Movies — filenames videoplayback.mp4 / .wav / .mp3

On Windows, embedded MP4 may fail depending on decoders; use **TRUCK_PACKER_LOOP** or a **.wav** override if needed.
